First release of **tlacuilo** — a pure-Go toolkit for TLA+ (*tlacuilo* is Nahuatl for "scribe").

```
go get github.com/aburan28/tlacuilo@v0.1.0
```

No dependencies beyond the Go standard library; Java + `tla2tools.jar` only where TLC actually runs (`tlc.EnsureJar` fetches it in one call).

## Highlights

- **TLA+ parser and canonical formatter** — the module language and full constant/action/temporal expression language, with SANY-compatible column-aware junction lists and the complete operator precedence table. All 10 standard modules shipped inside `tla2tools.jar` parse and round-trip stably.
- **Fluent builder** for writing specs from Go; everything it emits is parseable by the library, SANY, and TLC.
- **TLC integration** — config generation/parsing, jar discovery/download, streaming `-tool` protocol parsing into typed results, exit-code → status mapping, and counterexample traces (including liveness lassos and stuttering) decoded into typed TLA+ values with ITF JSON interchange.
- **Trace validation (`tracecheck`)** — validate Go implementations against specs: annotate actions (`tla:"var"` struct tags + `Recorder` calls), record under a deterministic test harness, and let TLC judge conformance; divergences are reported with the exact failing step and action.
- **Kubernetes controller verification template** (`examples/k8scontroller`) — an embedded spec with machine-checked safety *and* liveness proofs, a controller-runtime-shaped reconciler, a deterministic harness, a demonstrated divergence catch, and a dedicated **TLA+ proof** CI workflow that cannot silently skip.

## Verification behind this release

The tagged commit is green on both CI workflows, including live TLC runs: the exhaustive safety proof over all environment/reconcile interleavings, the quiescent-convergence liveness proof, and the trace-validation suite (reconcile controller, bounded queue, lock discipline — each with a buggy variant caught at the exact expected step).

## Known limitations

The TLA+2 proof language and PlusCal are out of scope (as in most tooling outside TLAPS); formatting is canonical rather than byte-preserving; `-workers auto` and the `CHECK_DEADLOCK` config keyword need a current TLC jar. See the guide's limitations section for the full list.
